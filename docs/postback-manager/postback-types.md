## Introduction

A postback URL is a specific type of URL used to send data from one server to another after a particular event occurs, often used in affiliate marketing. When a user completes a specified action (such as a purchase, form submission, or signup), the different postback types can be triggered to notify the affiliate or tracking platform that the conversion happened.

The postback URL typically includes parameters that carry dynamic data, such as the transaction ID, user ID, or sale amount, to ensure proper tracking of conversions or events. It's crucial in tracking systems to verify and record successful conversions.

## Selecting affiliate postback types

With Integr8, you may set up various [affiliate postback](https://integr8.co/docs/affiliate-postback-setup/ "Publisher Postback Setup") types by accessing the ***Postbacks*** menu. The postback types available are:

* **Click** (CPC)
* **Conversion** (as first event or unique event)
* **Event** (extra post conversion events)

### Important Note:

> In Integr8 platform, the conversion is always consider as the absolute first event and it has to be recorded in order to be able to track any other post-conversions events. ***There can not be an "Event" without a previously recorded "Conversion"***.
